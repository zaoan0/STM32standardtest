# STM32 直线循迹小车项目

## 项目描述

基于 STM32F103C8 的直线循迹小车系统，使用5路灰度传感器进行线路检测，电机PWM控制速度调节。

## 硬件配置

| 组件 | 说明 |
|------|------|
| MCU | STM32F103C8 |
| 传感器 | 5路灰度传感器（GPIOB: Pin 3/4/5/12/13） |
| 电机驱动 | 双电机PWM控制 |
| 串口 | USART1（PA9-TX，PA10-RX）115200bps |

## 功能模块

### 核心模块
- **motor.c/h** - 电机初始化和PWM控制
- **grayscale.c/h** - 灰度传感器读取
- **pid.c/h** - PID控制算法（预留）
- **main.c** - 主循迹逻辑

### 支持模块
- **encoder.c/h** - 编码器读取（可选）
- **trace.c/h** - 示踪引脚控制（可选）
- **debug.c/h** - 串口调试支持

## 版本历史

### v1.0.1（2026-04-26）— 当前版本 ✅

**核心机制：双阶段脱线恢复**

脱线后通过 `STABLE_COUNT` 稳定计数确认信号可靠再恢复，防止传感器抖动误触发。

**脱线恢复流程：**
```
全部传感器脱线
    ↓
进入脱线模式（lost_flag=1），保持上次速度运动
    ↓
任意一个传感器检测到线 → 开始稳定计数
    ↓
连续稳定 STABLE_COUNT 次 → 恢复正常循迹
```

**参数配置：**
```c
#define SPD_BASE      200   // 基础速度
#define SPD_SLIGHT     75   // 微调转向速度差
#define SPD_STRONG    150   // 强转向速度差
#define STABLE_COUNT   15   // 脱线恢复稳定计数（约0.5s）
```

**版本特点：**
- ✅ 稳定计数防止传感器抖动误触发恢复
- ✅ 脱线时保持上次转向速度，主动寻线
- ✅ 适合复杂赛道和弯道较多场景
- ⚠️ 恢复需稳定持续约0.5s，响应稍慢于v1.0.0

---

### v1.0.0（2026-04-26）— 历史版本

**核心机制：编译期可配置脱线模式**

通过 `LOST_MODE` 宏在编译时选择脱线策略，无需修改逻辑代码。

**模式说明：**
```c
#define LOST_MODE   0   // 保持上一次转向状态继续寻线（推荐）
#define LOST_MODE   1   // 直走等待重新找到线
```

**参数配置：**
```c
#define SPD_BASE    200
#define SPD_SLIGHT   75
#define SPD_STRONG  150
#define LOST_MODE     0   // 0 或 1
```

**版本特点：**
- ✅ 代码简洁，逻辑清晰，便于学习理解
- ✅ 两种脱线策略可通过宏一键切换
- ✅ 脱线响应速度快
- ⚠️ 无稳定计数，传感器抖动时可能误触

## 使用说明

1. 用 Keil 打开 `stm32standardtest.uvprojx`
2. 编译并下载到 STM32F103C8
3. 小车上电后自动开始循迹
4. 可通过串口（115200bps）监测调试信息
