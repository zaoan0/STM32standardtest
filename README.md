# STM32 直线循迹小车

这是我学习 STM32 嵌入式开发过程中做的一个直线循迹小车项目，基于 STM32F103C8 标准外设库，用5路灰度传感器检测黑线，双电机 PWM 控制转向。代码比较基础，记录了从简单到逐步完善的过程。

---

## 硬件配置

| 组件 | 说明 |
|------|------|
| MCU | STM32F103C8 |
| 传感器 | 5路灰度传感器（GPIOB: Pin 3/4/5/12/13） |
| 电机驱动 | 双电机 PWM 控制 |
| 串口 | USART1（PA9-TX，PA10-RX）115200bps |

## 版本历史

### v1.0.1（2026-04-26）— 当前版本 ✅

在 v1.0.0 基础上新增了**脱线状态标志位**和**稳定计数机制**，让脱线后的寻线恢复判断更可靠。

**相比 v1.0.0 新增的内容：**
- 新增 `lost_flag` 标志位，脱线时进入独立的恢复模式，正常循迹和脱线处理逻辑完全分离
- 新增 `stable_cnt` 稳定计数，传感器重新检测到线后需连续稳定 `STABLE_COUNT` 次才算真正找回，防止偶发抖动误判为找到线
- 脱线期间持续保持上次转向速度运动，主动向线的方向靠近

**脱线恢复逻辑：**
```
全部传感器脱线
    ↓
lost_flag = 1，保持上次左右速度继续运动
    ↓
仅有1个传感器检测到线 → stable_cnt 开始累加
    ↓
stable_cnt >= STABLE_COUNT（连续稳定）→ lost_flag = 0，恢复正常循迹
    ↓
期间传感器数量变化 → stable_cnt 清零重新计数
```

**参数配置：**
```c
#define SPD_BASE      200   // 基础速度
#define SPD_SLIGHT     75   // 微调转向速度差
#define SPD_STRONG    150   // 强转向速度差
#define STABLE_COUNT   15   // 恢复所需稳定帧数（约0.5s，可调小加快响应）
```

**已知问题：**
- ⚠️ 脱线后有时会一直转圈找不到线，即使小车已经回到线上，`stable_cnt` 仍然无法累积到 `STABLE_COUNT`
- 原因初步判断：判断条件为「仅1个传感器检测到线」过于严格，小车回归时可能同时触发2个传感器，导致条件不满足、计数一直被清零
- 后续计划放宽判断条件（如改为「至少1个传感器检测到线」），或降低 `STABLE_COUNT` 数值

### v1.0.0（2026-04-26）— 历史版本

最初版本，循迹逻辑和脱线处理写在同一个 while 循环里，结构简单，便于理解。通过 `LOST_MODE` 宏在编译时切换两种脱线策略。

**脱线处理方式：**
```c
#define LOST_MODE   0   // 模式0：保持上次转向速度继续寻线（推荐）
#define LOST_MODE   1   // 模式1：直走，等待重新碰到线
```

**参数配置：**
```c
#define SPD_BASE    200
#define SPD_SLIGHT   75
#define SPD_STRONG  150
#define LOST_MODE     0
```

**特点：**
- ✅ 代码量少，逻辑直观，适合入门学习
- ✅ 两种脱线策略一键切换，方便调试对比
- ⚠️ 没有独立的脱线状态，脱线处理和正常循迹混在一起
- ⚠️ 无稳定计数，传感器抖动可能误判为找回线

## 使用方法

1. 用 Keil 打开 `stm32standardtest.uvprojx`
2. 编译并烧录到 STM32F103C8
3. 小车上电后自动开始循迹
4. 如需调整效果，修改 `SPD_BASE`、`SPD_SLIGHT`、`SPD_STRONG` 参数重新编译
